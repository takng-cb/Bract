## 検証の考え方

- LLM は非決定的なため、**返答文の字面ではなく「検証済みの構造」**（推論されたブック、
  条件のフィールド・演算子・値の範囲、関連先、エラー文言）に期待値を置く。
- 件数はデモデータ依存のため原則検証しない（構造が正しければ一覧と同じ SQL で実行される）。
- AI のゆらぎで「直接確定」ではなく「候補提示」になるケースは、期待ブックが候補に
  含まれていれば選択して続行し、`(許容)` として記録する（エラーにはしない）。
- レート制限（アプリ側 20回/分/ユーザー＋プロバイダ側）を避けるため呼び出し間に待機を入れる。

## 関連する既知の制約

- AI 作成の対象は typed 7種（取引先/人物/車両/部品/物件/ToDo/活動）＋経費＋カスタムブック。
  商談は対象外（フォーム作成を推奨）、人材の案件は専用ウィザード（/quick/staffing）。
- URL 取り込みは公開 Web ページのみ（短縮共有リンク・ログイン必須ページは不可→本文コピペを案内）。
- 関連先の自動セットは ToDo/活動のみ（linkable）。照合先は取引先/人物/商談。
- 日本語の文章品質・固有名詞の安定性はプロバイダ依存（Groq llama < Gemini/Claude）。
  返答文の決定化（REQ-0064）と固有名詞ガード（textGuard）で主要事故は吸収済み。
- Groq 無料枠は 10万トークン/日。テストの連続実行や本番運用では有料プラン or Gemini/Anthropic を推奨。
