# ポストモーテム テンプレート

Sev1 / Sev2 障害発生後に必ず作成。Sev3 は任意。GitHub Issue に貼って残す。

---


---

## ポストモーテムの目的

- 個人を責めるためではなく、**システムとプロセスの改善**のため
- 同じクラスの障害を二度と起こさないための学習機会
- 顧客への透明性ある説明資料の元ネタ

## 過去の事例

- Issue #5 物件詳細の二重表示（cleanup 済み）
- Issue #6 取引先・商談詳細の 404（マイグレ全 Neon 適用ポリシーで対応）
- Issue #7 Vercel ビルド型エラー
- Issue #20 list ページ hydration 不全（調査中）
