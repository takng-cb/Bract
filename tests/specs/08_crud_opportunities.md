# 商談 CRUD テスト仕様書
---

## TC-OPP-001 新規作成

| 項目 | 内容 |
|------|------|
| 前提 | ログイン済み、商談一覧画面を表示している。紐付ける取引先レコードが存在する |
| 手順 | 1. 「新規作成」ボタンをクリックする<br>2. 商談名を入力する<br>3. 取引先（account）を選択する<br>4. ステージ（stage）を選択する<br>5. 金額（amount）を入力する<br>6. 確度（probability）を入力する<br>7. 完了予定日（close_date）を設定する<br>8. 「保存」ボタンをクリックする |
| 期待結果 | 商談が正常に作成され、詳細ページへリダイレクトされる。入力した全フィールドが正しく表示される |

---

## TC-OPP-002 必須項目チェック

| 項目 | 内容 |
|------|------|
| 前提 | ログイン済み、新規商談作成フォームを表示している |
| 手順 | 1. 商談名（name）を空のままにする<br>2. 「保存」ボタンをクリックする |
| 期待結果 | バリデーションエラーが表示され、「商談名は必須です」などのエラーメッセージが name フィールドの近くに表示される。レコードは作成されない |

## TC-OPP-003 詳細ページ表示

| 項目 | 内容 |
|------|------|
| 前提 | 活動履歴・タスクが紐付いた商談レコードが存在する |
| 手順 | 1. 商談一覧から対象レコードをクリックして詳細ページを開く |
| 期待結果 | 基本情報（商談名・取引先・ステージ・金額・確度・完了予定日など）と、関連する活動履歴セクション・タスクセクションがすべて正しく表示される |

## TC-OPP-004 編集

| 項目 | 内容 |
|------|------|
| 前提 | 既存の商談レコードの詳細ページを表示している |
| 手順 | 1. 「編集」ボタンをクリックする<br>2. 金額や確度などのフィールドを変更する<br>3. 「保存」ボタンをクリックする |
| 期待結果 | 変更内容が保存され、詳細ページに更新後の値が正しく表示される |

---

## TC-OPP-005 ステージ変更

| 項目 | 内容 |
|------|------|
| 前提 | ステージが「prospecting」の商談の詳細ページを表示している |
| 手順 | 1. ステージフィールドを編集またはクイックアクションでステージを変更する<br>2. prospecting → qualification → proposal → negotiation の順に進め、最終的に closed_won または closed_lost を選択して保存する |
| 期待結果 | 各ステップでステージが正常に更新される。closed_won または closed_lost に設定されたとき、商談が完了状態として表示される |

---

## TC-OPP-006 削除

| 項目 | 内容 |
|------|------|
| 前提 | 既存の商談レコードの詳細ページを表示している |
| 手順 | 1. 「削除」ボタンをクリックする<br>2. 確認ダイアログで「削除する」を選択する |
| 期待結果 | レコードが削除され、商談一覧ページへリダイレクトされる。削除された商談は一覧に表示されない |

---

## TC-OPP-007 金額フォーマット

| 項目 | 内容 |
|------|------|
| 前提 | 金額が設定された商談レコードが存在する（例：5000000）|
| 手順 | 1. 商談の詳細ページを表示する<br>2. 商談一覧でも金額列を確認する |
| 期待結果 | 金額フィールドが円記号付きカンマ区切りでフォーマットされて表示される（例：¥5,000,000）。一覧・詳細の両方で同じフォーマットが適用される |

---

## TC-OPP-008 関連活動

| 項目 | 内容 |
|------|------|
| 前提 | 複数の活動が紐付いた商談レコードの詳細ページを表示している |
| 手順 | 1. 詳細ページの「活動履歴」セクションを確認する |
| 期待結果 | 紐付いているすべての活動（種別・件名・日時・担当者など）が一覧表示される。活動の行をクリックすると活動詳細へ遷移するか、インライン展開される |
