# Tasks テスト仕様書
---

## TC-TSK-001 新規作成

| 項目 | 内容 |
|------|------|
| 手順 | 1. 「タスク」一覧ページへ移動する<br>2. 「新規作成」ボタンをクリックする<br>3. タイトル（title）に「提案書を送付する」と入力する<br>4. 期限日（due_date）を翌営業日に設定する<br>5. 優先度（priority）を「高」に設定する<br>6. 関連取引先・担当者・商談を必要に応じて選択する<br>7. 「保存」ボタンをクリックする |
| 期待結果 | タスクが正常に作成され、一覧ページにタイトル・期限日・優先度が表示される |

---

## TC-TSK-002 完了チェック

| 項目 | 内容 |
|------|------|
| 前提 | 少なくとも1件のタスクが作成済みで、done=falseの状態であること |
| 手順 | 1. タスク一覧ページでタスク行のチェックボックスをクリックして完了にする<br>2. 完了状態になったことを確認する<br>3. 再度チェックボックスをクリックして未完了に戻す<br>4. タスク詳細ページを開き、詳細ページ上の完了ボタン/チェックボックスでも同様に完了→未完了のトグルを行う |
| 期待結果 | 一覧・詳細ページのどちらからでもdone/undoneのトグルが正常に動作する。完了状態のタスクはチェック済みの表示になる |

## TC-TSK-003 優先度表示

| 項目 | 内容 |
|------|------|
| 前提 | 優先度「高」「中（通常）」「低」の3種類のタスクが作成済みであること |
| 手順 | 1. タスク一覧ページを開く<br>2. 各タスクの優先度バッジを確認する |
| 期待結果 | 優先度「高」のタスクには赤色または「緊急」を示すバッジが表示される。「中」「低」は異なる色またはラベルで区別される |

## TC-TSK-004 期限切れ表示

| 項目 | 内容 |
|------|------|
| 前提 | due_dateが過去の日付で、done=falseのタスクが作成済みであること |
| 手順 | 1. タスク一覧ページを開く<br>2. 期限切れタスクの期限日の表示を確認する |
| 期待結果 | 期限切れの未完了タスクの期限日が赤色または強調表示される。完了済みタスクには期限切れ表示が適用されない |

## TC-TSK-005 期限別グルーピング

| 項目 | 内容 |
|------|------|
| 前提 | 「今日」「今週」「それ以降」「期限なし」に該当する複数のタスクが作成済みであること |
| 手順 | 1. タスク一覧ページを開く<br>2. タスクのグループ分けを確認する |
| 期待結果 | タスクが「今日」「今週」「それ以降」「期限なし」のセクションに分けて表示される。各セクションには対応する期限のタスクのみが含まれる |

## TC-TSK-006 編集

| 項目 | 内容 |
|------|------|
| 前提 | 少なくとも1件のタスクが作成済みであること |
| 手順 | 1. タスク詳細ページを開く<br>2. 「編集」ボタンをクリックする<br>3. タイトルを変更する<br>4. 期限日を変更する<br>5. 優先度を変更する<br>6. 「保存」ボタンをクリックする |
| 期待結果 | 変更内容が正常に保存され、詳細ページおよび一覧ページに更新後の値が表示される |

---

## TC-TSK-007 削除

| 項目 | 内容 |
|------|------|
| 前提 | 少なくとも1件のタスクが作成済みであること |
| 手順 | 1. タスク詳細ページを開く<br>2. 「削除」ボタンをクリックする<br>3. 確認ダイアログで「OK」をクリックする |
| 期待結果 | タスクが削除され、一覧ページに該当レコードが表示されなくなる |

---

## TC-TSK-008 取引先詳細からの確認

| 項目 | 内容 |
|------|------|
| 前提 | 特定の取引先に紐付いたタスクが複数作成済みであること |
| 手順 | 1. 取引先一覧ページへ移動する<br>2. タスクが紐付いている取引先の詳細ページを開く<br>3. 関連タスクセクションを確認する |
| 期待結果 | 取引先詳細ページの関連タスクセクションに、その取引先に紐付いたタスクがすべて表示される。完了状態・優先度・期限日が正しく表示される |
