# 担当者 CRUD テスト仕様書
---

## TC-CON-001 新規作成（法人担当者）

| 項目 | 内容 |
|------|------|
| 前提 | ログイン済み、担当者一覧画面を表示している。紐付ける取引先レコードが存在する |
| 手順 | 1. 「新規作成」ボタンをクリックする<br>2. 担当者種別（contact_type）を「法人担当者」に設定する<br>3. 氏名・メールアドレスなどの必須項目を入力する<br>4. 取引先（account）フィールドで既存の取引先を選択する<br>5. 「保存」ボタンをクリックする |
| 期待結果 | 担当者が正常に作成され、詳細ページへリダイレクトされる。contact_type が「法人担当者」、選択した取引先がリンクとして表示される |

## TC-CON-002 新規作成（個人顧客）

| 項目 | 内容 |
|------|------|
| 前提 | ログイン済み、担当者一覧画面を表示している |
| 手順 | 1. 「新規作成」ボタンをクリックする<br>2. 担当者種別（contact_type）を「個人顧客」に設定する<br>3. 氏名・メールアドレスなどの必須項目を入力する<br>4. 取引先フィールドは空のまま「保存」ボタンをクリックする |
| 期待結果 | 担当者が正常に作成される。contact_type が「個人顧客」と表示され、取引先の紐付けなしで保存できる |

## TC-CON-003 タブ切り替え

| 項目 | 内容 |
|------|------|
| 前提 | 法人担当者・個人顧客の両種別の担当者レコードが複数存在する |
| 手順 | 1. 担当者一覧画面を表示する<br>2. 「法人担当者」タブをクリックする<br>3. 次に「個人顧客」タブをクリックする |
| 期待結果 | 「法人担当者」タブでは contact_type=business のレコードのみ表示される。「個人顧客」タブでは contact_type=consumer のレコードのみ表示される。タブ切り替え時に一覧が即座に絞り込まれる |

## TC-CON-004 詳細ページ表示

| 項目 | 内容 |
|------|------|
| 前提 | 活動履歴が紐付いた担当者レコードが存在する |
| 手順 | 1. 担当者一覧から対象レコードをクリックして詳細ページを開く |
| 期待結果 | 基本情報（氏名・メール・電話・担当者種別・取引先リンクなど）と、関連する活動履歴セクションがすべて正しく表示される |

## TC-CON-005 編集

| 項目 | 内容 |
|------|------|
| 前提 | 既存の担当者レコードの詳細ページを表示している |
| 手順 | 1. 「編集」ボタンをクリックする<br>2. 電話番号や役職などのフィールドを変更する<br>3. 「保存」ボタンをクリックする |
| 期待結果 | 変更内容が保存され、詳細ページに更新後の値が正しく表示される |

---

## TC-CON-006 削除

| 項目 | 内容 |
|------|------|
| 前提 | 既存の担当者レコードの詳細ページを表示している |
| 手順 | 1. 「削除」ボタンをクリックする<br>2. 確認ダイアログで「削除する」を選択する |
| 期待結果 | レコードが削除され、担当者一覧ページへリダイレクトされる。削除された担当者は一覧に表示されない |

## TC-CON-007 取引先へのリンク

| 項目 | 内容 |
|------|------|
| 前提 | 取引先が紐付いた法人担当者の詳細ページを表示している |
| 手順 | 1. 詳細ページの取引先フィールドに表示されている取引先名のリンクをクリックする |
| 期待結果 | 紐付いている取引先の詳細ページへ遷移する |

## TC-CON-008 テキスト名刺インポート

| 項目 | 内容 |
|------|------|
| 前提 | ログイン済み、担当者一覧画面または新規作成画面を表示している |
| 手順 | 1. 「名刺インポート」または「テキストから取り込む」ボタンをクリックし TextImportModal を開く<br>2. 名刺のテキスト情報（氏名・会社名・役職・メール・電話など）をテキストエリアに貼り付ける<br>3. 「解析」または「インポート」ボタンをクリックする<br>4. 解析結果を確認し、必要に応じて修正して「保存」する |
| 期待結果 | 入力されたテキストから各フィールドが自動抽出され、フォームに反映される。確認・保存後に担当者レコードが作成される |
