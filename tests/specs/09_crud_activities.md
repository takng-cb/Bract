# Activities テスト仕様書
---

## TC-ACT-001 新規作成（コール）

| 項目 | 内容 |
|------|------|
| 手順 | 1. 「活動」一覧ページへ移動する<br>2. 「新規作成」ボタンをクリックする<br>3. 種別（type）に「コール」を選択する<br>4. 件名（subject）に「架電：初回ヒアリング」と入力する<br>5. 本文（body）に通話内容の詳細を入力する<br>6. 実施日時（occurred_at）を入力する<br>7. 関連取引先に既存の取引先を選択する<br>8. 「保存」ボタンをクリックする |
| 期待結果 | 活動レコードが正常に作成され、一覧ページにtype=callの活動が表示される。件名・取引先・実施日時が正しく保存されている |

---

## TC-ACT-002 新規作成（メール/ミーティング/メモ）

| 項目 | 内容 |
|------|------|
| 前提 | TC-ACT-001が完了済みであること |
| 手順 | 1. 「新規作成」ボタンをクリックする<br>2. 種別に「メール」を選択し、件名・本文・実施日時を入力して保存する<br>3. 再度「新規作成」から種別に「ミーティング」を選択し、件名・本文・実施日時を入力して保存する<br>4. 再度「新規作成」から種別に「メモ」を選択し、件名・本文・実施日時を入力して保存する |
| 期待結果 | メール・ミーティング・メモの3種別それぞれで活動レコードが正常に作成される。コールと合わせ4種別（call/email/meeting/note）すべて一覧に表示される |

## TC-ACT-003 担当者の複数紐付け

| 項目 | 内容 |
|------|------|
| 前提 | 複数の担当者（コンタクト）が登録済みであること |
| 手順 | 1. 「新規作成」または既存の活動編集画面を開く<br>2. 担当者（contacts）フィールドで1人目の担当者を選択する<br>3. さらに2人目、3人目の担当者を追加選択する<br>4. 「保存」ボタンをクリックする |
| 期待結果 | 活動レコードに複数の担当者が紐付いて保存される。詳細ページで全担当者が表示される |

## TC-ACT-004 詳細ページ表示

| 項目 | 内容 |
|------|------|
| 前提 | 少なくとも1件の活動レコードが作成済みであること |
| 手順 | 1. 活動一覧ページへ移動する<br>2. 任意の活動レコードの件名リンクをクリックする |
| 期待結果 | 活動詳細ページが表示され、種別・件名・本文・実施日時・関連取引先・担当者がすべて正しく表示される |

---

## TC-ACT-005 編集

| 項目 | 内容 |
|------|------|
| 前提 | 少なくとも1件の活動レコードが作成済みであること |
| 手順 | 1. 活動詳細ページを開く<br>2. 「編集」ボタンをクリックする<br>3. 件名を変更する<br>4. 本文を更新する<br>5. 「保存」ボタンをクリックする |
| 期待結果 | 変更内容が正常に保存され、詳細ページに更新後の値が表示される |

---

## TC-ACT-006 削除

| 項目 | 内容 |
|------|------|
| 前提 | 少なくとも1件の活動レコードが作成済みであること |
| 手順 | 1. 活動詳細ページを開く<br>2. 「削除」ボタンをクリックする<br>3. 確認ダイアログで「OK」をクリックする |
| 期待結果 | 活動レコードが削除され、一覧ページに該当レコードが表示されなくなる |

---

## TC-ACT-007 取引先詳細からの確認

| 項目 | 内容 |
|------|------|
| 前提 | 特定の取引先に紐付いた活動レコードが複数作成済みであること |
| 手順 | 1. 取引先一覧ページへ移動する<br>2. 活動が紐付いている取引先の詳細ページを開く<br>3. 活動タイムラインまたは関連活動セクションを確認する |
| 期待結果 | 取引先詳細ページのタイムラインに、その取引先に紐付いた活動がすべて表示される。実施日時の新しい順に並んでいる |
