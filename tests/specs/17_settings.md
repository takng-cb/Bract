# 設定テスト仕様書

## 対象機能
- 個人設定（表示名・パスワード）
- 列表示設定
- ナビゲーション順序リセット
- システム設定（パスワードポリシー・セッションタイムアウト・自己登録）

---

## TC-SET-001 表示名変更

| 項目 | 内容 |
|------|------|
| 前提 | 任意のロールでログイン済み |
| 手順 | 1. `/settings` を開く<br>2. 表示名フィールドを新しい名前に変更する<br>3. 「保存」をクリックする |
| 期待結果 | 設定が保存され、ページ再読み込み後も変更した表示名が反映されている |

---

## TC-SET-002 パスワード変更

| 項目 | 内容 |
|------|------|
| 前提 | メール＋パスワードで登録したアカウントでログイン済み（Google OAuthアカウントは対象外） |
| 手順 | 1. `/settings` を開く<br>2. 現在のパスワードを入力する<br>3. 新しいパスワードと確認用パスワードを入力する<br>4. 「変更する」をクリックする |
| 期待結果 | パスワードが変更され、ログアウト後に新しいパスワードでログインできる |

---

## TC-SET-003 列表示設定の保存

| 項目 | 内容 |
|------|------|
| 前提 | 任意のロールでログイン済み |
| 手順 | 1. 取引先一覧を開く<br>2. 列表示設定を開き、特定の列のチェックを外す<br>3. 設定を保存する<br>4. ページを再読み込みする |
| 期待結果 | チェックを外した列が非表示のまま維持されており、他のユーザーの列設定には影響しない |

---

## TC-SET-004 ナビ順序（個人）リセット

| 項目 | 内容 |
|------|------|
| 前提 | 個人用ナビ順序をシステムデフォルトから変更済み |
| 手順 | 1. 個人設定のナビゲーションセクションを開く<br>2. 「システムデフォルトに戻す」をクリックする<br>3. 確認ダイアログで「リセット」をクリックする |
| 期待結果 | 個人用のナビ順序設定が削除され、システムデフォルトのナビ順序で表示される |

---

## TC-SET-005 パスワードポリシー設定

| 項目 | 内容 |
|------|------|
| 前提 | 管理者としてログイン済み |
| 手順 | 1. `/settings` のセキュリティセクションを開く<br>2. 最小文字数および複雑さの要件（大文字・数字・記号など）を設定する<br>3. 「保存」をクリックする<br>4. 要件を満たさないパスワードで変更を試みる |
| 期待結果 | 設定が保存され、ポリシーを満たさないパスワードはエラーで拒否される |

---

## TC-SET-006 セッションタイムアウト設定

| 項目 | 内容 |
|------|------|
| 前提 | 管理者としてログイン済み |
| 手順 | 1. `/settings` のセキュリティセクションを開く<br>2. セッションタイムアウト時間（例：30分）を設定する<br>3. 「保存」をクリックする |
| 期待結果 | 設定が保存される。設定した時間を超えて操作がない場合、次のアクセス時にログイン画面へリダイレクトされる |

---

## TC-SET-007 自己登録の有効／無効

| 項目 | 内容 |
|------|------|
| 前提 | 管理者としてログイン済み |
| 手順 | 1. `/settings` のユーザー管理セクションを開く<br>2. 自己登録のトグルを「無効」に切り替える<br>3. 「保存」をクリックする<br>4. ログアウトして `/signup` または登録ページへアクセスする |
| 期待結果 | 自己登録が無効の場合、登録フォームが表示されない、またはエラーが返される |
