# TC-18: 関係性管理（多対多リレーション）

## 概要

`relationship_definitions` + `relationship_values` による汎用多対多リレーションシステムのテスト。  
管理者が関係性定義を作成し、詳細ページから関連レコードを追加・削除・閲覧する。

---

## 前提条件

- 管理者アカウントでログイン済み
- 物件レコードが 1 件以上存在する（property_id = A）
- 商談レコードが 1 件以上存在する（opportunity_id = B）

---

## TC-18-01: 関係性定義の一覧表示

**操作**: サイドバー > 「🔗 関係性管理」をクリック  
**URL**: `/admin/relationships`

**期待結果**:
- [ ] ページが表示される
- [ ] 初期シード「物件・商品 ⟷ 商談 — 関連商談 / 関連物件」が表示される
- [ ] 各定義に「多対多」バッジが表示される
- [ ] 「＋ 新規定義」ボタンが存在する
- [ ] 「削除」ボタンが存在する

---

## TC-18-02: 関係性定義の作成

**操作**: `/admin/relationships/new` に遷移し、フォームを入力して保存

**入力**:
- ソース: 取引先（accounts）
- ターゲット: 担当者（contacts）
- ラベル: 「関連担当者」
- 逆ラベル: 「関連取引先」
- 種別: 多対多

**期待結果**:
- [ ] `/admin/relationships` にリダイレクト
- [ ] 新しい定義「取引先 ⟷ 担当者 — 関連担当者」が一覧に表示される

---

## TC-18-03: 関係性定義のバリデーション

**操作**: 必須項目（ソース / ターゲット / ラベル）を空欄で保存

**期待結果**:
- [ ] フォームが送信されない（ブラウザの required バリデーション）

---

## TC-18-04: 物件詳細ページで「関連商談」セクションが表示される

**操作**: `/properties/{A}` を開く

**期待結果**:
- [ ] 「🔗 関連商談」セクションが表示される
- [ ] 「商談」バッジが表示される
- [ ] 関連レコードがない場合「関連レコードがありません」と表示される
- [ ] 追加フォーム（ID 入力欄 + 追加ボタン）が表示される（editor 以上）

---

## TC-18-05: 物件詳細ページから関連商談を追加

**前提**: TC-18-04 完了

**操作**: 「関連商談」セクションのID入力欄に商談 B の ID を入力し「追加」をクリック

**期待結果**:
- [ ] ページがリロードされ商談 B が「関連商談」一覧に表示される
- [ ] 商談 B の名前がリンクとして表示される
- [ ] リンクをクリックすると `/opportunities/{B}` に遷移する

---

## TC-18-06: 商談詳細ページで逆方向の関連が表示される

**前提**: TC-18-05 完了（物件 A と商談 B を関連付け済み）

**操作**: `/opportunities/{B}` を開く

**期待結果**:
- [ ] 「関連レコード」セクションが表示される
- [ ] 「🔗 関連物件」セクションが表示される（reverse_label）
- [ ] 物件 A が一覧に表示される
- [ ] 「解除」ボタンが表示される（editor 以上）

---

## TC-18-07: 関連レコードの解除

**前提**: TC-18-05 完了

**操作**: 商談詳細ページの「🔗 関連物件」セクションで物件 A の「解除」をクリック → 確認ダイアログで「OK」

**期待結果**:
- [ ] 確認ダイアログが表示される
- [ ] 解除後、物件 A が「関連物件」一覧から消える
- [ ] 物件 A の詳細ページ（`/properties/{A}`）でも「関連商談」一覧から商談 B が消える

---

## TC-18-08: 重複追加の防止

**前提**: 物件 A と商談 B が関連付け済み

**操作**: 再度、物件詳細ページから商談 B の ID を入力して「追加」

**期待結果**:
- [ ] 「すでに関連付けられています」エラーメッセージが表示される
- [ ] 重複レコードは作成されない

---

## TC-18-09: 関係性定義の削除（カスケード）

**前提**: 物件 A と商談 B の関係性定義に紐づく `relationship_values` が存在する

**操作**: `/admin/relationships` から「物件・商品 ⟷ 商談」定義の「削除」をクリック → 確認ダイアログで「OK」

**期待結果**:
- [ ] 定義が一覧から消える
- [ ] `/properties/{A}` で「関連商談」セクションが非表示になる
- [ ] `relationship_values` の関連データも自動削除される（ON DELETE CASCADE）

---

## TC-18-10: 閲覧者権限での制限

**前提**: 閲覧者（viewer）でログイン済み

**操作**: `/properties/{A}` を開く

**期待結果**:
- [ ] 「関連商談」セクションは表示される
- [ ] ID 入力フォームおよび「追加」ボタンは非表示
- [ ] 「解除」ボタンは非表示

---

## TC-18-11: 関係性管理ページは管理者のみ表示

**前提**: 編集者（editor）でログイン済み

**操作**: サイドバーを確認

**期待結果**:
- [ ] 「🔗 関係性管理」リンクがサイドバーに表示されない
- [ ] `/admin/relationships` に直接アクセスすると 403 / リダイレクトされる
